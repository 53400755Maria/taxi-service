# Taxi Service

Сервис для заказа такси с CI/CD

---

## Описание

Веб-приложение для заказа такси (одностраничное приложение, SPA). Пользователь может просматривать список доступных водителей, создавать заказы, отслеживать их статус и управлять заказами.

Приложение реализовано на Python с использованием микрофреймворка Flask и предоставляет REST API для взаимодействия с клиентской частью.

**Реализованные endpoints:**

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/` | Главная страница (UI) |
| GET | `/api/drivers` | Получение списка водителей |
| GET | `/api/orders` | Получение списка заказов |
| POST | `/api/orders` | Создание нового заказа |
| PUT | `/api/orders/<id>` | Обновление статуса заказа |
| GET | `/api/health` | Проверка работоспособности |

## Технологии

- **Python 3.11** — язык программирования
- **Flask** — веб-фреймворк
- **pytest** — тестирование
- **pytest-cov** — оценка покрытия кода тестами
- **flake8** — линтер (проверка стиля кода)
- **black** — автоматическое форматирование кода
- **Docker** — контейнеризация приложения
- **docker-compose** — оркестрация контейнеров для локального запуска
- **GitHub Actions** — CI (непрерывная интеграция)
- **Render** — CD (непрерывное развертывание)

---

## Запуск локально (без Docker)

Для запуска приложения на локальной машине необходимо:

1. Клонировать репозиторий
2. Создать и активировать виртуальное окружение Python
3. Установить зависимости из `app/requirements.txt`
4. Скопировать `.env.example` в `.env` и при необходимости отредактировать переменные
5. Запустить приложение командой `flask run`

После запуска приложение будет доступно по адресу `http://localhost:5000`

---

## Запуск в Docker

Для запуска приложения в контейнеризованном окружении используется Docker Compose:

1. Из корневой директории проекта выполнить команду `docker compose up --build`
2. Дождаться сборки образа и запуска контейнера
3. Приложение станет доступно по адресу `http://localhost:5000`

Остановка контейнера выполняется командой `docker compose down`

## CI/CD пайплайн

### Непрерывная интеграция (CI)

**Триггер:** автоматический запуск при создании Pull Request в ветку `main`.

**Выполняемые проверки:**

| Шаг | Инструмент | Описание |
|-----|------------|----------|
| **Линтинг** | `flake8`, `black` | Проверка стиля кода (макс. длина строки 100 символов) и форматирования |
| **Тестирование** | `pytest`, `pytest-cov` | Запуск модульных тестов с отчётом о покрытии кода |
| **Сборка Docker-образа** | `docker build` | Проверка, что приложение успешно собирается в контейнер |

**Защита ветки `main`:**
- Pull Request не может быть вмёржен, если хотя бы одна проверка завершилась с ошибкой
- Требуется прохождение всех статусов: `Lint Code`, `Run Tests`, `Build Docker Image`
- Все изменения вносятся только через Pull Request

### Непрерывное развертывание (CD)

**Триггер:** автоматический запуск при слиянии Pull Request в ветку `main`.

Платформа **Render** отслеживает изменения в ветке `main` и автоматически:
- Клонирует репозиторий
- Собирает Docker-образ
- Развертывает контейнер
- Перезапускает приложение

Новая версия становится доступной по публичному URL через 2–3 минуты после слияния.

## Переменные окружения

| Переменная | Описание | Пример значения |
|------------|----------|-----------------|
| `FLASK_APP` | Точка входа в приложение | `app.server` |

| `PYTHONUNBUFFERED` | Отключение буферизации вывода (для логов) | `1` |
| `PORT` | Порт для запуска сервера | `5000` |
